📊 시각화 설명

🎯 노드(Node)의 의미:

  • 형광 노랑 노드: 시장을 주도하는 핵심 테마 (중동전쟁, 에너지, 인프라 등)
  • 색상 노드: 개별 주식 종목 (테마에 따라 색상 분류)
  • 노드 크기: 상대적 중요도 또는 시장 영향력

🔗 연결선(Link)의 의미:

  • 선의 길이: 주식과 테마의 관련도 (짧을수록 강한 연관)
  • 선의 굵기: 영향 강도를 나타냄 (굵을수록 강함)
  • 다중 선 연결: 여러 테마에 영향받는 종목 = 테마 간의 '교점' 역할

✨ 핵심 인사이트:

  • 비슷한 색상 종목들이 한곳에 모여 있음 = 같은 테마로 묶임
  • 테마 중간에 위치한 종목 = 두 테마의 영향을 모두 받음 (포트폴리오 분산에 유리)
  • 고립된 종목 = 특정 테마에만 강하게 의존

✅ Force Network의 강점

자동 최적 배치

물리 엔진이 노드들을 자동으로 배치하여, 관련도 높은 종목들이 자연스럽게 가까워짐. 수동으로 배치할 필요 없음.

다차원 관계 표현

테마-종목, 종목-종목 관계를 동시에 시각화. 스프레드시트나 차트로는 불가능한 복잡한 관계를 직관적으로 표현.

상호작용성

드래그로 노드를 움직이고, 호버로 상세 정보 확인. 실시간으로 관계 구조를 탐색할 수 있음.

Cluster 자동 형성

코딩 없이 같은 테마 종목들이 자동으로 한 그룹을 이룸. 포트폴리오 다각화 전략 수립에 직접 활용 가능.

교점(Hub) 자동 발견

여러 테마와 연결된 종목이 자동으로 중심에 위치. 이 종목들은 시장 변화에 민감한 '척도' 역할.

시간 기반 분석

링크 강도(value)를 실시간으로 업데이트하면 시간에 따른 종목 관계 변화를 동적으로 표현 가능.

⚠️ Force Network의 한계

확장성 문제

노드 수가 100개를 넘으면 시각화가 복잡해지고 성능 저하 발생. 대규모 주식 데이터(수천 개 종목)에는 부적합.

비결정적 배치

매번 실행할 때마다 노드 위치가 다름. 일관된 레이아웃이 필요한 분석에는 부적합.

정량적 분석 어려움

시각적 거리와 실제 관련도가 완벽하게 일치하지 않음. 정확한 수치 분석은 다른 도구 필요.

초기 조건 민감성

Force 파라미터(반발력, 연결거리 등)에 따라 결과가 크게 달라짐. 세팅이 까다로움.

시계열 데이터 미지원

정적 스냅샷만 보여줌. 종목 가격 추이나 수익률 같은 시계열 정보는 별도 표현 필요.

모바일 최적화 부족

대규모 캔버스라 모바일에서 상호작용이 불편. 반응형 디자인 구현이 복잡.
