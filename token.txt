

# 1️⃣ 전체 토큰 구조 (3종 토큰 시스템)

이 플랫폼은 사실 **토큰이 3개 존재하는 구조**가 가장 깔끔하다.

### ① 부동산 지분 토큰

예

```
A-Token
```

의미

```
부동산 수익증권 지분
```

예

```
건물 A = 10,000 토큰
```

---

### ② 임대료 지급 토큰

두 가지 선택지가 있다.

### 방법1 (가장 현실적)

임대료를 **USDT**로 납부

구조

```
청년
↓
USDT 구매
↓
스마트컨트랙트 전송
↓
투자자 자동 분배
```

---

### 방법2 (플랫폼 토큰)

ASTO가 자체 **Rent Token** 발행

예

```
ASTO-R
```

구조

```
청년
↓
ASTO-R 구매
↓
컨트랙트 전송
↓
투자자 분배
```

하지만 현실적으로는

**USDT가 훨씬 신뢰도가 높다.**

---

# 2️⃣ 스마트컨트랙트 전체 구조

실제 구현은 **5개 컨트랙트**가 필요하다.

---

## 1️⃣ Property Contract

부동산 정보 저장

예

```
부동산 주소
총 가치
토큰 수
청년 지분
투자자 지분
```

---

## 2️⃣ Ownership Token Contract

부동산 토큰 발행

보통

```
ERC20
```

또는

증권형

```
ERC3643
```

표준 사용

---

## 3️⃣ Rental Payment Contract

여기가 핵심이다.

임대료 처리

구조

```
청년
↓
USDT 전송
↓
Rental Contract
↓
투자자 지갑 자동 분배
```

스마트컨트랙트 기능

```
receiveRent()
distributeRent()
```

---

## 4️⃣ Liquidation Contract

임대료 미납 처리

구조

```
미납 발생
↓
청년 토큰 일부 감소
↓
투자자 증가
```

예

```
청년 20%
→
18%
```

---

## 5️⃣ Oracle Contract

현실 정보 연결

예

```
임대료 납부 확인
소송 상태
퇴거 여부
```

이걸 블록체인에 넣는다.

---

# 3️⃣ 임대료 흐름 (중요)

이 구조가 사업의 핵심이다.

### ① 청년이 임대료 납부

```
청년
↓
거래소
↓
USDT 구매
```

---

### ② 스마트컨트랙트 전송

```
청년
↓
Rental Contract
USDT 전송
```

---

### ③ 자동 분배

스마트컨트랙트가 실행

```
USDT
↓
투자자 지갑
```

예

```
투자자1 5%
투자자2 10%
투자자3 2%
```

자동 지급

---

# 4️⃣ 미납 발생 구조

예

```
월 임대료
100 USDT
```

청년이 못 냈다.

스마트컨트랙트

```
30일 대기
↓
부분청산
```

예

```
청년 지분 20%
↓
19.8%
```

이 지분이 투자자들에게 분배된다.

---

# 5️⃣ 매각 발생

건물 매각 시

```
매각금
↓
신탁사
↓
ASTO
↓
스마트컨트랙트
↓
토큰 비율 분배
```

투자자

```
USDT
```

또는

```
법정화폐
```

로 지급 가능

---

# 6️⃣ 플랫폼에서 가장 중요한 UX

청년이 직접 USDT를 사는 건 불편하다.

그래서 보통 이렇게 한다.

```
청년
↓
ASTO 앱
↓
카드 결제
↓
자동 USDT 구매
↓
스마트컨트랙트 납부
```

즉

**청년은 그냥 임대료 낸다고 생각하면 된다.**

뒤에서 블록체인이 돌아간다.

---

# 7️⃣ 이 구조의 진짜 강점

이 모델이 기존 부동산과 다른 이유는 이것이다.

기존

```
임대료
→ 집주인
```

이 모델

```
임대료
→ 수백명 투자자 자동 분배
```

즉

**임대료 금융화**

이다.
